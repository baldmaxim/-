# WORKFLOW: РАСЦЕНКА КЛАДОЧНЫХ РАБОТ
## Пошаговый алгоритм от парсинга ВОР до финального QA

**Версия:** 1.0 | **Дата:** 2026-02-20  
**Применение:** Обязательный алгоритм для Sonnet-worker при расценке раздела кладки

---

## ОБЗОР ЭТАПОВ

```
ЭТАП 1: ПАРСИНГ ВОР          → Список ВСЕХ позиций кладки
    ↓
ЭТАП 2: КЛАССИФИКАЦИЯ        → Материал, толщина, тип (нар/внутр/шахт)
    ↓
ЭТАП 3: AVR ПОИСК            → Цены-кандидаты из базы
    ↓
ЭТАП 4: КОНВЕРТАЦИЯ ЕДИНИЦ   → Проверка м²/м³, пересчёт
    ↓
ЭТАП 5: ЦЕНООБРАЗОВАНИЕ      → Финальная цена с надбавками
    ↓
ЭТАП 6: SANITY CHECK         → Проверка через price_matrix.json
    ↓
ЭТАП 7: QA (НЕЗАВИСИМЫЙ)     → qa_checklist.md
    ↓
РЕЗУЛЬТАТ: pricing.json
```

## ЭТАП 1: ПАРСИНГ ВОР — НАЙТИ ВСЕ ПОЗИЦИИ

### 1.1 Сканирование
- Пройти ВСЕ строки ВОР (все разделы, все листы)
- Для каждой строки проверить ключевые слова:
  ```
  ВКЛЮЧИТЬ: кирпич, газобетон, газосиликат, ПГП, пазогребнев, 
  КСЦ, КСР, керамзитобетон, кладка, блок+стен, блок+перегород,
  полнотелый, Ytong, Bonolit, POROTHERM, поризованный камень
  
  ИСКЛЮЧИТЬ: клинкерная плитка, керамогранит, стяжка+керамзит,
  фахверк (→ металл), утеплитель, облицовка
  ```

### 1.2 Проверить ВСЕ разделы
- **Раздел 1.x** — первый этаж / подвал / подземная часть
- **Раздел 1.7.x** — дополнительные кладочные работы 1 этажа
- **Раздел 2.x** — типовые этажи (ОГРОМНЫЕ ОБЪЁМЫ!)
- **Раздел 2.3.x** — кладочные стены типовых этажей
- **Раздел 2.6.x** — прочие работы (могут содержать кладку)
- **Раздел 3.x+** — кровельный этаж, технический этаж

### 1.3 Агрегация объёмов
- Суммировать объём по ВСЕМ колонкам (секции/этажи)
- Если объём в колонке 10 = 0, проверить колонки 11-20
- Записать единицу: м² или м³

### 1.4 Результат этапа
Файл: `01_vor_raw.json` с полями:
```json
{
  "position_id": "2.3.3.4",
  "description": "Газобетон D600 200мм внутренний",
  "vor_unit": "м3",
  "volumes_by_section": [588.9, 626.8, 617.8, ...],
  "total_volume": 4573.0,
  "vor_section": "2.3.3"
}
```

## ЭТАП 2: КЛАССИФИКАЦИЯ ПОЗИЦИЙ

### 2.1 Определить для каждой позиции:
| Параметр | Как определить | Пример |
|----------|---------------|--------|
| Материал | Ключевые слова | "газобетон D500" |
| Толщина, мм | Число в описании: "200мм", "в полкирпича"=120мм | 200 |
| Плотность | Dxxx: D400, D500, D600 | D500 |
| Тип стены | "наружн"→нар, "внутр/МОП/межкварт"→внутр, "шахт"→шахт | внутренняя |
| Назначение | Перегородка (<150мм) / стена (≥150мм) / несущая (250+мм) | стена |
| Особенности | EI45, К0, влагост., звукоизол. | EI45 |
| Этажность | Раздел 1.x=1этаж, 2.x=типовые | типовые |

### 2.2 Пересчёт единиц ВОР
- Если ВОР в м³, пересчитать в м²: `area_m2 = volume_m3 / thickness_m`
- Если ВОР в м², оставить как есть
- ЗАПИСАТЬ оба значения!

### 2.3 Результат этапа
Файл: `02_classified.json` — дополненный `01_vor_raw.json`

## ЭТАП 3: AVR ПОИСК

### 3.1 Формирование запроса
Для каждой позиции:
1. Составить поисковый запрос: `{материал} {толщина}мм кладка {тип}`
2. Добавить синонимы из ENCYCLOPEDIA_FINAL.md (раздел 3)
3. Запустить AVR поиск

### 3.2 Интерпретация результатов
- **sim ≥ 0.70**: хорошее совпадение, но ПРОВЕРИТЬ ЕДИНИЦЫ
- **sim 0.60–0.69**: среднее, использовать с осторожностью
- **sim < 0.60**: плохое, НЕ использовать — fallback на price_matrix.json
- **ЛЮБОЙ sim**: проверить единицу AVR (м²/м³)

### 3.3 Записать решение
```json
{
  "avr_price": 11872,
  "avr_unit": "куб.м",
  "avr_sim": 0.76,
  "avr_source": "ЖК Суворов-Парк"
}
```

## ЭТАП 4: КОНВЕРТАЦИЯ ЕДИНИЦ (КРИТИЧЕСКИЙ!)

### 4.1 Алгоритм определения реальной единицы AVR

```
ВХОД: avr_price, avr_unit_label, material, thickness_mm

1. Загрузить range из price_matrix.json для material+thickness
2. Вычислить:
   - price_as_m2 = avr_price
   - price_as_m3_converted = avr_price × (thickness_mm / 1000)
3. Проверить:
   - fits_m2 = range.min×0.7 ≤ price_as_m2 ≤ range.max×1.3
   - fits_m3 = range.min×0.7 ≤ price_as_m3_converted ≤ range.max×1.3
4. Решение:
   - fits_m2 и НЕ fits_m3 → цена уже в м²
   - fits_m3 и НЕ fits_m2 → цена в м³, конвертировать
   - Оба подходят → взять ближе к typical
   - Ни один → FALLBACK на typical из матрицы
```

### 4.2 Примеры
| AVR цена | Метка | Материал | Толщина | Как м² | Как м³→м² | Решение |
|----------|-------|----------|---------|--------|-----------|---------|
| 11,872 | куб.м | газобетон | 200мм | 11,872 ❌>7000 | 2,374 ✅ | Это м³! |
| 4,848 | куб.м | кирпич | 120мм | 4,848 ✅ | 582 ❌ | Это м² |
| 2,424 | куб.м | газобетон | 100мм | 2,424 ✅ | 242 ❌ | Это м² |
| 2,424 | куб.м | ПГП | 80мм | 2,424 ✅ | 194 ❌ | Это м² |

### 4.3 Записать конвертацию
```json
{
  "detected_unit": "м3",
  "conversion": "11872 × 0.20 = 2374",
  "base_price_m2": 2374,
  "decision_reason": "AVR цена 11872 > red_flag 7000 для газобетон 200мм → конвертация из м³"
}
```

## ЭТАП 5: ЦЕНООБРАЗОВАНИЕ

### 5.1 Базовая цена
Выбрать ЛУЧШУЮ из доступных:
1. **AVR** (если sim ≥ 0.65 и единицы проверены) → base = avr_converted
2. **Энциклопедия** → base = калькуляция (материал + работа + расходники)
3. **Матрица** → base = typical из price_matrix.json

### 5.2 Правило "верхняя треть"
```
final_base = range.min + (range.max - range.min) × 0.67
```
Или: корректировка AVR цены в верхнюю треть диапазона матрицы.

### 5.3 Надбавки
Определить применимые коэффициенты:
- Наружная стена? → ×1.05–1.10
- Шахта? → ×1.10–1.20
- EI45/EI60? → ×1.10–1.20
- Высота >3м? → ×1.05–1.15
- Межквартирная? → ×1.05–1.10

**МАКСИМУМ суммарных надбавок: ×1.35**

### 5.4 Финальная цена
```
final_price = final_base × Π(коэффициенты)
```

Проверка: `range.min ≤ final_price ≤ range.red_flag_above`

### 5.5 Записать решение
```json
{
  "position": "1.1.2.1",
  "base_source": "AVR converted + encyclopedia cross-check",
  "base_price_m2": 2374,
  "adjustments": "upper_third → 3500",
  "surcharges": [],
  "final_price_m2": 3500,
  "total": 1856120
}
```

## ЭТАП 6: SANITY CHECK (САМОПРОВЕРКА)

### 6.1 Пройти по каждой позиции:
- [ ] Цена в пределах range из price_matrix.json?
- [ ] Цена не выше red_flag_above?
- [ ] Цена не ниже red_flag_below?

### 6.2 Глобальная проверка:
- [ ] Средневзвешенная цена в пределах 3,100–5,000 руб/м²?
- [ ] Общий объём разумен для проекта?
- [ ] Позиции >20% бюджета проверены тройным стандартом?

### 6.3 При обнаружении проблем:
- Записать в лог
- Пересчитать позицию
- НЕ финализировать до исправления

---

## ЭТАП 7: НЕЗАВИСИМЫЙ QA

- Передать pricing.json агенту QA (Opus)
- QA проверяет по qa_checklist.md (все 5 проверок)
- Если QA вернул ❌ → исправить и повторить
- Если QA вернул ✅ → финализировать
